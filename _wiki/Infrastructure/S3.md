# 🪣 S3

> [[Home]] > [[Infrastructure Index|Infrastructure]] > S3

Two S3 buckets are provisioned, both with a random hex suffix for uniqueness.

---

## Images Bucket — `photoshare-images-<hex>`

Stores uploaded photo objects under the `photos/` prefix.

| Setting | Value |
|---|---|
| Public ACLs | Allowed (`block_public_acls = false`) |
| Public bucket policy | Allowed |
| Public read | `s3:GetObject` on `arn:.../photos/*` |
| CORS | GET, PUT, POST from `*`; expose `ETag`; max-age 3000s |
| Object key format | `photos/<uuid>.<ext>` |

> ⚠️ Images are **publicly readable** without authentication. Only write/delete require auth (via Lambda + pre-signed URLs).

### Pre-signed URL

Generated by `upload_url.py` via `boto3`:
```python
s3.generate_presigned_url(
    "put_object",
    Params={"Bucket": BUCKET_NAME, "Key": object_key, "ContentType": content_type},
    ExpiresIn=3600,   # PRESIGN_EXPIRY env var
)
```

---

## Frontend Bucket — `photoshare-frontend-<hex>`

Hosts the static SPA. Served through CloudFront (not directly).

| Setting | Value |
|---|---|
| Website config | `index_document = index.html`, `error_document = index.html` |
| Public read | `s3:GetObject` on `arn:.../*` |
| CloudFront origin | Uses S3 website endpoint (`http-only` origin protocol) |

### Files deployed

```
index.html          (no-cache)
styles.css          (max-age 1y)
js/config.js        (no-cache — runtime config injected at deploy)
js/api.js
js/auth.js
js/upload.js
js/gallery.js
js/ui.js
js/app.js
```

---

## IAM Access

The Lambda role has scoped permissions:
```
s3:PutObject, s3:GetObject, s3:DeleteObject
Resource: arn:aws:s3:::photoshare-images-<hex>/*
```
